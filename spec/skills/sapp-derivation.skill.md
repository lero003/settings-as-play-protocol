# SaPP Derivation Skill

## Purpose

This Skill Document defines how AI assistants should create, read, and validate SaPP derivations.

A derivation is a new setting document or creative output that intentionally changes, extends, adapts, or reinterprets an existing SaPP setting.

## Canonical Principle

Derivation should preserve origin and explain change.

An AI assistant should make clear:

- what the derivative came from
- what stayed the same
- what changed
- why the change is allowed or useful
- how the original author or source should be credited

## Required Derivation Information

A derivation should include:

- Derived From
- Original By
- What Stayed
- What Changed
- Public Range
- Non-public Range
- Legal License / Permission
- Canonical Status
- Cultural Agreement
- Credit Notes

## AI Derivation Procedure

When asked to derive from a SaPP setting, an AI assistant should:

1. Read the source SaPP Core.
2. Identify Core Rules that must remain stable.
3. Identify Variables that may be changed.
4. Check Derivation & Credit boundaries before generating.
5. Ask for missing permission or boundary details when publication is likely.
6. Propose changes inside the Variables first.
7. Mark any Core Rule change as a new branch, alternate version, or incompatible derivative.
8. Preserve source attribution.
9. Produce a derivative card or updated SaPP document.

## Compatibility Levels

Use these labels when helpful:

- Compatible derivative: preserves Core Rules and changes only Variables or module details.
- Alternate version: changes one or more Core Rules intentionally.
- Inspired-by work: uses broad influence but should not imply continuity or compatibility.
- Non-canonical derivative: may be useful as a fan, test, or visual reference, but is not part of the source setting.

## Conflict Handling

If a requested derivation conflicts with the source Core Rules, the assistant should state the conflict and offer one of:

- revise the request to stay compatible
- create an alternate version
- create an inspired-by work with clearer separation
- stop if permission or boundary issues block the request

## Non-Goals

This Skill Document does not decide legal reuse rights. It only defines how SaPP records derivation, boundaries, and attribution.

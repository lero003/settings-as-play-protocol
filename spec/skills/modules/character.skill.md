# Character Module Skill

## Purpose

Use this module when a SaPP setting needs character-specific information.

The Character Module should help an AI assistant preserve a character's recognizable identity across images, scenes, dialogue, games, stories, and derivative settings.

## Sections

### Appearance

Physical form, clothing, colors, silhouette, age impression, species, accessories, and distinctive features.

Appearance should be concrete enough for repeated visual or descriptive use.

### Personality

Temperament, speech style, habits, reactions, values, contradictions, and emotional tendencies.

Personality should guide behavior, not only list adjectives.

### Relationship

Important relationships, social role, distance from others, bonds, conflict, dependency, or rivalry.

### Change Pattern

Rules for how the character changes across time, place, emotion, season, event, medium, or derivation.

Change Pattern should connect to SaPP Core Variables when possible.

### Do / Do Not

Recommended and avoided portrayals.

Use this section for sensitive boundaries, tone boundaries, and identity-preserving constraints that are too specific for SaPP Core.

## Validation Notes

Warn if Appearance contradicts Core Rules.

Warn if Personality is too vague to guide dialogue or behavior.

Warn if Do / Do Not silently changes legal, public, or non-public boundaries that belong in Derivation & Credit.

# Game Module Skill

## Purpose

Use this module when a SaPP setting needs interactive rules, player role, repeatable play, simulation, progression, or game adaptation.

The Game Module should help an AI assistant translate a setting into playable structure while preserving the setting's identity and boundaries.

## Sections

### Player Role

Who the player is inside the setting.

### Goal

What the player tries to achieve, understand, protect, change, survive, restore, or discover.

### Mechanics

Actions, rules, systems, resources, choices, constraints, verbs, and feedback.

### Loop

The repeated play cycle.

Example: explore -> choose -> change -> reflect.

### Win / Lose / Change Conditions

Conditions for success, failure, transformation, branching, state change, or continuation.

### Progression

Unlocks, growth, decay, memory, route changes, difficulty changes, relationship changes, or world changes.

## Validation Notes

Warn if Mechanics reward behavior forbidden by Core Rules or Do / Do Not boundaries.

Warn if Win / Lose conditions imply a moral or canonical conclusion that the setting does not support.

Warn if the player role requires permissions or public range beyond what Derivation & Credit allows.
